# v0.1.1 - cron timeout baseline fix

## Fixed

- installer now writes `agents.defaults.timeoutSeconds=900` when the target OpenClaw config does not define one
- existing explicit timeout settings are preserved
- long-running cron jobs and agent turns no longer get cut off by the shorter built-in OpenClaw default after installation

## Notes

- this is a patch release focused on reliability
- no profile layout changes
- no memory schema changes
