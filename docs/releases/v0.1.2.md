# v0.1.2 - sub-agent announce timeout fix

## Fixed

- installer now writes `agents.defaults.subagents.announceTimeoutMs=300000` together with `agents.defaults.timeoutSeconds=900` and `agents.defaults.subagents.runTimeoutSeconds=900`
- long-running cron jobs no longer fail early when sub-agents complete work but the completion announce step exceeds OpenClaw's shorter built-in default
- installer tests now verify both sub-agent runtime and announce timeout defaults

## Notes

- this is a patch release focused on cron reliability
- no profile layout changes
- no memory schema changes
- no local-memory transport timeout changes
